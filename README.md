## 프로젝트 소개

### 1. 프로젝트 설명

**프로젝트 로고**:

![Logo_black](https://github.com/Demopeu/TWM/assets/156268475/47bc562f-f5db-42b7-992d-27c5fdc3f784)

**프로젝트 이름**: TWM(Travel With Movies)

**프로젝트 개요**:
TWM(Travel With Movies)은 사용자가 다양한 나라의 영화를 감상하면서 각 나라의 문화와 감성을 경험 할 수 있도록 알맞은 영화를 추천하는 프로젝트입니다. 

**프로젝트 기간**: 2024.05.08 ~ 2024.05.23

![jara](https://github.com/Demopeu/TWM/assets/156268475/58875e41-ad55-4dce-a6c3-e8805770af3b)

**프로젝트 상세 설명**:
TWM(Travel With Movies)은 자체 개발한 알고리즘을 통해 사용자에게 다양한 나라의 영화를 추천합니다. 이 프로젝트는 단순한 영화 감상을 넘어, 영화를 통해 각 나라의 문화, 역사, 사회적 배경 등을 이해할 수 있는 기회를 제공합니다.

**주요 기능 및 목표:**

- **다양한 영화 추천**: 세계 각국의 영화를 추천하여 사용자에게 다양한 문화적 경험을 선사합니다.

- **문화 이해 증진**: 추천된 영화를 통해 사용자는 각 나라의 고유한 문화와 감성을 배울 수 있습니다.

- **여행 욕구 자극**: 영화를 통해 각국의 아름다운 풍경과 문화를 접하면서, 해당 나라로의 여행을 꿈꾸게 만듭니다.

- **맞춤형 알고리즘**: 사용자의 선호도를 분석하여 개인화된 영화 추천을 제공합니다. 

TWM 프로젝트는 사용자에게 문화적 다양성을 경험하게 하고, 영화라는 매개체를 통해 세계 여러 나라의 문화를 더욱 깊이 이해하고 느낄 수 있도록 하는 것을 목표로 합니다.

### 2. 시작 가이드

#### 요구 사항

- Node.js 12 이상
- Vue.js 3.4.21 이상
- axios 1.6.8 이상
- bootstrap 5.3.3 이상
- pinia 2.1.7 이상
- Python 3.9 이상
- Django 4.2.8 이상
- 기타 필요한 패키지 및 라이브러리

---

#### 설치 및 실행

1. 저장소를 클론합니다:

    ```bash
    git clone https://github.com/Demopeu/TWM.git
    ```

2. 백엔드 폴더로 이동합니다:

    ```bash
    cd ../final_pjt_back/
    ```

3. 가상 환경을 설정하고 필요한 패키지를 설치합니다:

    ```bash
    python -m venv venv
    venv\Scripts\activate  # 윈도우의 경우
    pip install -r requirements.txt
    ```

4. 백엔드 프로젝트 루트에 `.env` 파일을 생성하고 필요한 환경 변수를 설정합니다.:

    ```
    SECRET_KEY=your_secret_key_here
    DATABASE_URL=your_database_url_here
    ```
 
5. 데이터베이스 마이그레이션을 수행합니다:

    ```bash
    python manage.py migrate
    ```

6. 초기 데이터를 로드합니다:

    ```bash
    python manage.py loaddata users.json movies.json articles.json comments.json
    ```

7. 백엔드 서버를 실행합니다:

    ```bash
    python manage.py runserver
    ```

8. 프론트엔드 폴더로 이동합니다:

    ```bash
    cd ../final_pjt_front/
    ```

7. 필요한 패키지를 설치합니다:

    ```bash
    npm install
    ```

8. 프론트엔드 서버를 실행합니다:

    ```bash
    npm run dev
    ```

9. 웹 브라우저에서 `http://localhost:8080`을 열고 애플리케이션을 확인합니다.

## 프로젝트 결과물

### 1. 핵심 알고리즘 소개

#### **PPAP 알고리즘(Pre Processing and After Processing Algorithm)**

1. **영화 데이터 불러오기**
    - 사용자가 특정 대륙이나 나라를 선택하면, 해당 국가에서 제작된 영화를 데이터베이스(DB)에서 가져옵니다.
    - 그 후에는 사용자의 위시리스트 목록을 불러옵니다.

2. **영화 목록 필터링**
    - 사용자가 이미 본 영화나 위시리스트에 담은 영화는 추천 목록에서 제외됩니다.
    - 필터링된 영화 목록을 편의상 'Pre Processing Products(PPP)'라고 부릅니다.

3. **위시리스트와 본 영화 목록 관리**
    - 사용자는 위시리스트 목록에서 본 영화 목록으로 이동할 수 있습니다.
    - 본 영화 목록이 업데이트되면, 해당 영화의 장르들이 사용자의 선호 장르 필드에 저장됩니다.

4. **선호 장르 데이터 저장 방식**
    - 예를 들어, 사용자가 로맨스와 액션 장르가 포함된 영화를 시청했다면, 해당 장르들은 사용자의 선호 장르 필드에 저장됩니다.

5. **2차 필터링: 선호 장르 기반 추천**
    - 사용자의 선호 장르가 2개 이하일 경우, PPP에서 무작위로 50개의 영화를 추출하여 추천합니다.
    - 사용자의 선호 장르 데이터가 3개 이상일 경우, 선호 장르를 기반으로 가중치를 적용합니다.

6. **가중치 적용 및 영화 리스트 정리**
    - PPP 내의 모든 영화에 가중치를 적용하고, 이를 기준으로 영화를 4개의 집단으로 나누어 정렬합니다.
    - 동일한 영화가 반복되지 않도록 각 집단을 셔플합니다.

7. **최종 추천 영화 목록 구성**
    - 가중치가 매겨진 영화 리스트를 'After Processing Products(APP)'라고 부릅니다.
    - 최종적으로 추천할 때는 최대 50개의 영화를 추출해 추천합니다.
    - APP가 50개 이하일 경우, 모든 영화를 추천합니다.
    - APP가 50개 이상일 경우, 가중치가 높은 영화를 우선적으로 고려합니다.

8. **결과**
    - 사용자는 가중치가 높은 영화부터 최대 50개의 영화를 추천받습니다.

### 2. 목표 서비스 구현 및 실제 구현 정도

- **서비스 목표**:

    - 사용자의 선호도를 분석하여 세계 각국의 영화를 추천하는 맞춤형 알고리즘을 통해, 영화를 감상하면서 다양한 문화와 역사를 이해하고 여행 욕구를 자극하게 만들자.

- **공통 사항**:

    - nav바에 게시판 이동, 프로필 이동, 로그아웃, 뒤로가기 기능을 제공하여 편의성을 높임
    - 인증된 사용자만 서비스를 이용하도록 제한
 
- **로그인 page**:

    - 목표 서비스
      - 배경에 세계지도 모양을 배치하여 여행과 관련된 웹사이트임을 강조
      - 파스텔톤 색상을 사용하여 장난스럽고 편안한 분위기를 제공
      - 아이디와 비밀번호 입력 시 로그인 구현
      - 편의를 위해 비밀번호 찾기 기능 구현

    - 실제 구현
      - 아이디,비밀번호,이메일 모두 입력 시 로그인 구현
      - 로그인 한 유저는 로그인 페이지 진입 불가
      - 비밀번호 확인 불일치, 이메일 오류 등등 다양한 경우 오류 메세지 출력
      - 비밀번호 찾기 기능은 추후 개발 예정
     
- **나라 선택 page**:

    - 목표 서비스
      - 유명하고, 접근성이 높은 국가를 선정하여 유저의 호기심을 자극
      - 화살표를 사용한 가시성 높은 디자인을 구상

    - 실제 구현
      - 반응형 사이트 제작을 위한 디자인 변경

- **영화 추천 page**:

    - 목표 서비스
      - 유저가 원하는 나라의 영화들을 영화 추천 알고리즘을 통해 추천
      - 원하는 영화의 정보 확인, API를 통해 공식 트레일러 접근 구현
      - 디테일 page에서 즉각적으로 게시판으로 이동할 수 있도록 구현

    - 실제 구현
      - 사용자의 선호 장르를 고려하고, 약간의 무작위성을 추가한 알고리즘 구현
      - 영화 최대 추천 개수를 50개로 제한
      - 디테일 page가 아닌, 위시리스트에서 수정 가능하도록 구현

- **커뮤니티 추천 page**:

    - 목표 서비스
      - 유저들이 다양한 국가들의 리뷰를 작성하고 소통할 수 있는 공간을 마련
      - 국가 혹은 대륙별 태그를 사용해 글을 분류
      - 태그 색깔을 포함해 전체적으로 파스텔톤을 채용하여 캐주얼한 느낌을 제공
      - 게시글 수정, 삭제를 구현
      - 댓글 삭제 구현
      - 게시글 검색 구현

    - 실제 구현
       - 현재 보고 있는 태그가 강조되도록 구현
       - 게시글 최신순으로 갱신
       - 게시글의 범주 구현
       - 게시글 상세 페이지에서 작성자 프로필로 이동할 수 있도록 구현
       - 게시글 검색은 추후 지원 예정

- **프로필 page**:

    - 목표 서비스
        - 유저의 인증서, 위시리스트, 본 영화를 소개할 수 있는 페이지를 구현
        - 특히 인증서는 핵심 요소이므로 눈에 띄는 트로피 모양으로 설정
        - 배경은 심플하고 모던하게 구성
        - 프로필 사진, 이메일 등 기본적인 유저 정보 출력
        - 시청 목록은 더보기를 통한 구현
        - 회원 정보 수정 구현

    - 실제 구현
        - 위시리스트와 시청 목록을 스크롤바로 직관적이고 한 눈에 볼 수 있도록 구현
        - 위시리스트 클릭으로 간단히 시청 목록으로 넘길 수 있도록 구현
        - 위시리스트 작동 도움말 제공
        - 회원 탈퇴 기능 구현
        - 회원 정보 수정은 추후 지원 예정
        - 현재 랜덤 프로필 이미지 기능만 제공되며 추후 유저 개인이 설정할 수 있도록 구현 예정

## 프로젝트 규칙

### 1. commit message

-**Commit message 구조 :**

```
type:title
ex. git commit -m "docs: README 내용 수정"
```

-**Commit message type 종류 :**

![type](https://github.com/Demopeu/TWM/assets/156268475/c31f78f6-67bb-46f2-a830-58590515a02e)

### 2. ommisions message

-**ommisions message 작성법 :**

1. **대주제 선택(ex. front, back)**

2. **직접적으로 문제가 있거나, 누락된 부분이 있는 페이지 작성**

    ```
    1. Login.vue
    - username, password 이모티콘 누락
    ```

    - 구동이 안될 정도의 큰 문제가 있을 경우, 대주제 바로 아래에 "##"를 이용하여 즉시 작성후 **!important**를 어미에 작성

    - 구동은 되나, 전체에 걸쳐서 오류가 있을 경우, 대주제 바로 아래에 "##"를 이용하여 즉시 작성

3. **"##" 에 작성한 내용은 merge 설명에 기입 후, MM혹은 카카오톡을 이용하여 즉시 알림**

4. **완료되거나 해결 된 이슈는 삭제하지 말고 취소선을 그은 이후, 대주제 completed로 이동**

### 3. pull request

1. **로컬에서 변경 사항 커밋**

    - 새로운 기능이나 수정한 내용을 로컬에서 커밋, 이 때, 각자의 branch 이용

    ```bash
    git add .
    git commit -m "docs: README 내용 수정"
    ```

2. **원격 저장소로 브랜치 push**

    - 로컬에서 변경한 내용을 원격 저장소로 push

    ```bash
    git push branchname
    ```

3. **Pull Request 생성**

    - GitHub 또는 GitLab 등의 원격 저장소에서 Pull Request를 생성
    - Pull Request에는 작업한 내용을 간결하게 요약하여 작성

4. **팀장의 승인 대기**

    - Pull Request를 작성한 후, 팀장에게 리뷰를 요청
    - 팀장은 변경 내용을 확인하고 필요한 경우 코멘트를 남기거나 승인을 진행

5. **Merge 충돌 없을 경우**

    - 변경 내용에 충돌이 없는 경우, 팀장이 승인하고 바로 Merge

6. **Merge 충돌 발생 시 중재**

    - 변경 내용에 충돌이 발생한 경우, 팀장이 충돌을 해결하고 Merge를 진행
    - 충돌을 해결한 후, 다시 승인을 요청하거나 직접 Merge

8. **Pull Request 완료**

    - Pull Request가 완료되면 해당 작업은 원격 저장소의 메인 브랜치에 반영

## 프로젝트 후기

- 김동현

    -**코드리뷰를 통한 이해력 상승**-

        - 처음으로 팀장 역할을 맡게 되어, 전체 프로젝트 관리 및 조율을 하게 되었습니다. 주요 기술 결정부터 merge 충돌 시, 코드 리뷰까지 처음 하는 일이라 시행착오가 많았습니다. 그래도, 손에 익어감에 따라 코드를 읽고 이해하는 속도가 빨라졌습니다. 이러한 경험이 협업 때 도움이 될 것이라고 느꼈습니다.

    -**방대한 프론트엔드 스킬 활용**-

        - 영화 프로젝트를 진행하면서, 프론트엔드 부분이 중요하다고 생각하여 따로 공부를 진행하였습니다. SSAFY에서 배운 내용부터, 배우지 않은 라이브러리까지 방대한 스킬을 익혀 사용할 수 있었다는 점이 이번 프로젝트의 가장 큰 의의였습니다.

    -**디자이너와 개발자 간의 협업점 파악**-
        
        - 디자이너가 없었기 때문에, 직접 피그마를 사용하여 디자인도 구상하며, 그것을 직접 Vue.js를 이용하여 구현하였습니다. CSS에 대하여 지식이 부족했던 부분을 채워줌과 동시에, 협업 시 배려할 부분과 필요충분조건에 대하여 알게 되었습니다.
    
    -**프로젝트 대한 깊이와 발표 능력 향상**-
    
        - 프로젝트를 완료한 이후, 프로젝트의 README를 작성하였습니다. 새로운 사용자가 이용할 때의 마음가짐을 생각하여, 프로젝트 기능을 상세하게 적었을 뿐만 아니라, 시작 가이드도 작성하였습니다. 이것을 바탕으로 발표를 진행하였습니다. 프로젝트의 장점을 강조해야 하며, 소개하는 법에 대하여 다시 한번 알 수 있게 되었습니다. 이것을 통하여 프로젝트를 설계할 때, 제작할 때, 발표할 때 더욱 세심하고 꼼꼼하게 진행할 수 있게 됐습니다. 

- 노재훈

    - 싸피에 와서 규모 있는 프로젝트는 처음 겪어보았다. 기간은 약 2주일이 주어졌고 신선한 아이디어를 뽑는 것에 자신이 없는 나로써는 창작에 대한 두려움이 컸다. 그러나 팀원과 브레인 스토밍을 통해 주제를 생각해내고 문제를 해결하는 과정에서 자신감을 얻었다.

    - 프로젝트를 진행하며 생각보다 많으 문제를 겪었는데, 먼저 github의 활용이었다. 평소 노션으로만 필기를 하고, 프로그래밍 협업을 할 기회가 전무하여 github를 본격적으로 처음 써봤는데 자주 merge 충돌을 일으키고 브랜치로 이동하지 않은 채 push를 하는 등의 문제를 겪었고 이를 해결하며 github에 익숙해져 갔다. 그 외에도 Jira를 통한 스케줄 관리의 필요성과 목업과 설계를 통한 튼튼한 모델링 구축의 중요성을 알게 되었다.

    - 팀원과의 의견 충돌 문제도 생각보다 많았다. 서로 원하는 디자인과 코딩 스타일, 그리고 성향이 달라서 약간의 불협화음도 있었으며 특히 components와 views의 구성방식, 알고리즘을 구현하는 방식에서 의견이 많이 갈렸다. 그 외에도 성향 차이가 나타나 문제 해결 방식 혹은 외부 프로그램을 사용하는 것에 대한 의견 마찰도 존재했다. 또한 종종 의사소통 문제를 겪기도 하였지만, 원활하고 꾸준한 소통으로 합의점을 찾고 문제를 해결해갔다. 프로젝트를 통해 협업에서 실시간 소통이 얼마나 중요한 것인지 깨닫게 되었다.

    - 개인주의적 성향이 미치는 영향도 적지 않다 생각했다. 항상 보고서나 일처리를 혼자서 하는 것이 일상이었고, 그 편을 더 선호하는 개인주의적 성향이 강한 성격인데 팀프로젝트에 부정적인 영향을 끼친다는 것을 직접 피부로 느꼈으며, 자신의 포지션을 적절히 조절해가며 협업을 진행해야 한다고 느꼈다.

    - 이번 프로젝트는 1학기 동안 SSAFY에서 배운 외부 API 사용 방법, Django로 서버를 구축하는 방법, Vue3로 서버와 소통하고 프론트를 구축하는 방법을 총동원해서 진행해야 했다. 이 과정에서 자신의 지식의 한계점을 뼈저리게 느꼈고, 부족한 점과 사회에 일원이 되어 맡은 역할을 훌륭히 수행해 나가는 개발자로서 성장하기 위해 배워야 할 점이 많다는 것을 깨닫게 되는 귀중한 시간이 되었다.
